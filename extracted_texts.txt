COMPANY OVERVIEW

About AstroTalk
AstroTalk is India's #1 online astrology and spiritual-tech platform, founded in Noida on October 18, 2017. It operates as a two-sided marketplace connecting users with verified astrologers for consultations via chat, voice, and video calls.

The platform offers Vedic astrology, tarot reading, numerology, Vastu, Kundli matching, palmistry, and more — available 24/7 across 60+ countries.

--- Slide 12 ---


SWOT ANALYSIS


STRENGTHS
•  80% market share in India's online astrology
•  45,000+ verified astrologers — largest network
•  Strong unit economics: LTV:CAC = 3:1
•  Profitable since FY22; ₹250 Cr profit FY25
•  50M+ downloads, strong brand recall


WEAKNESSES
•  High marketing dependency (25% of revenue)
•  Subjectivity in service quality — hard to standardize
•  Stigma around astrology limits word-of-mouth
•  FY25 net profit dipped 62% despite revenue growth
•  Heavy reliance on Meta/Google ad platforms


OPPORTUNITIES
•  Indian spiritual market: $150Bn by 2032
•  Global NRI community — high ARPU markets
•  D2C e-commerce barely scratched (₹140Cr in yr 1)
•  AI personalization & predictive astrology features
•  SEO organic growth still largely untapped


THREATS
•  265+ competitors; potential new well-funded entrants
•  Regulatory risk — currently unregulated sector
•  Ad platform algorithm changes can spike CAC
•  Legal challenges (CCI complaint, trademark cases)
•  Consumer trust scandals could damage reputation
--- Slide 13 ---
































KEY TAKEAWAYS
What makes AstroTalk a landmark case study in Indian spiritual-tech


01
Timing + Trust = Explosive Growth  —  AstroTalk solved the #1 problem in astrology — trust — through rigorous vetting, then grew 20X during COVID when demand spiked.


02
Freemium is a Powerful Growth Engine  —  "First Chat Free" dramatically lowered acquisition barriers. Combined with a 3:1 LTV:CAC ratio, it built a self-sustaining growth loop.


03
Diversification Mitigates Risk  —  Expanding from consultations → D2C e-commerce → offline stores reduces over-dependence on one revenue stream.


04
Profitable Growth Story  —  ₹250 Cr profit on ₹1,210 Cr revenue at age 8 — rare in Indian startups. The IPO path is validated by strong fundamentals.


05
Platform Network Effects at Scale  —  45K+ astrologers and 50M+ users create compounding network effects — more astrologers attract more users and vice versa.

HEALCONNECT
Healing  •  Vastu  •  Numerology  •  Wellness

Technology Stack Document
Architecture, Decisions & Rationale
Version 1.0  •  April 2025
Prepared for: HealConnect Development Team
Classification: Confidential

1. Executive Summary
HealConnect is a professional two-sided marketplace connecting users with verified practitioners in energy healing, Vastu Shastra, numerology, tarot, crystal therapy, sound healing, and related wellness disciplines. It is modelled on the Astrotalk consultation model, adapted for the holistic wellness space.
This document defines the complete technology stack selected for HealConnect and explains the rationale behind every major technology decision. The stack has been chosen to support three simultaneous applications — a user mobile app, a practitioner mobile app, and a web-based admin panel — on a 45-day MVP timeline with enterprise-grade scalability to 100,000+ concurrent users.
Every technology choice below prioritises four non-negotiable principles:
Speed to market — technologies with large talent pools, excellent documentation, and proven patterns for marketplace apps.
Scalability — horizontally scalable services that handle India-market growth from Day 1 without re-architecture.
Security — PCI-DSS alignment, OWASP Top 10 compliance, end-to-end encryption, and GDPR-ready data architecture.
Azure-native integration — the chosen stack integrates cleanly with Microsoft Azure (India Central), reducing operational complexity.

2. Architecture Overview
HealConnect follows a modern, decoupled three-tier architecture comprising a mobile presentation layer (React Native), a REST/WebSocket API backend (Node.js on Azure App Service), and a PostgreSQL relational data store on Azure's managed database service.
The real-time consultation core — the most critical component — is powered by Socket.IO backed by Azure Web PubSub for horizontal WebSocket scaling, and Agora.io for in-app audio/video calls. Payments are delegated to Razorpay for the Indian market (Phase 1) and Stripe for international expansion (Phase 3), ensuring PCI-DSS compliance without handling card data on HealConnect's servers.
Background workloads (payout processing, scheduled reminders, report generation) are handled by Azure Service Bus and Azure Functions — fully managed, serverless, and cost-effective at scale.

3. Mobile Applications
The mobile layer is the primary user touchpoint. HealConnect requires two native-quality mobile apps — a User App and a Practitioner App — both targeting iOS and Android simultaneously.
  📱  Mobile Framework  
3.1 React Native with Expo (TypeScript)
React Native is the definitive choice for HealConnect's mobile layer. It delivers native iOS and Android applications from a single TypeScript codebase, eliminating the cost and coordination overhead of maintaining two separate native codebases.
Why React Native over Flutter or native development:
Single codebase ships both the User App and Practitioner App on iOS and Android simultaneously — critical given the 45-day MVP constraint.
JavaScript/TypeScript ecosystem means the same language runs across mobile, backend (Node.js), and web (Next.js), enabling engineer context-switching without language barriers.
Expo managed workflow removes native build environment setup from developers' machines — critical for a fast-moving intern/startup team. OTA updates via Expo Updates allow hot-fixes without App Store resubmission.
The React Native ecosystem has first-class SDKs for every required third-party service: Agora.io, Razorpay, Firebase FCM, Socket.IO — all officially maintained.
Flutter, while performant, uses Dart — a language most web/backend developers don't know, slowing down full-stack engineers who need to contribute across the stack.


4. Backend API
The backend is the operational core of HealConnect — handling authentication, per-minute billing, real-time session management, wallet transactions, and practitioner payouts. It needs to be async-first, type-safe, and horizontally scalable.
  ⚙️  Server Runtime & Framework  
4.1 Node.js + TypeScript + Express.js
Node.js with TypeScript is the optimal runtime for HealConnect's backend for several compounding reasons:
Async-first architecture — Node.js's event loop is built for I/O-heavy workloads like WebSocket connections (thousands of concurrent sessions), database queries, and third-party API calls (Razorpay, Agora, Twilio). It does not block on I/O.
TypeScript eliminates an entire category of runtime errors — null reference exceptions, type mismatches, and incorrect API contract usage — that are catastrophic in financial and real-time systems.
JSON-native runtime — JSON is the wire format for all APIs. Node.js parses and serialises JSON without type conversion overhead.
Shared language with frontend — backend and frontend engineers can review each other's code, share validation schemas (Zod), and share type definitions via a monorepo.
Express.js is the battle-tested, minimal web framework with the largest middleware ecosystem. For a startup-speed build, its simplicity beats NestJS's overhead.


5. Database & Storage
  🗄️  Primary Database  
5.1 PostgreSQL — Azure Database for PostgreSQL Flexible Server
PostgreSQL is the correct primary database for HealConnect. As a financial marketplace, HealConnect handles wallet transactions, per-minute billing debits, practitioner payouts, and payment records — all of which require ACID-compliant, relational integrity that NoSQL databases cannot provide reliably.
ACID transactions — wallet balance updates, billing debits, and payout calculations must be atomic. A failed mid-transaction cannot leave a user's wallet in a partial state.
Foreign key constraints and relational integrity — sessions reference users and practitioners; transactions reference sessions and wallets. Referential integrity is enforced at the database level, not just the application layer.
JSON/JSONB columns — for semi-structured data like practitioner certification details and user wellness profiles, PostgreSQL's JSONB provides flexibility without sacrificing query performance.
Azure's managed Flexible Server provides automated backups, point-in-time restore, read replicas, and high-availability with standby — zero infrastructure management overhead.
Why not MongoDB: A document database would force application-layer transaction management for financial operations, increasing complexity and the surface area for bugs in billing logic.


6. Real-Time, Communications & Payments
  💬  Real-Time Chat  
6.1 Socket.IO + Azure Web PubSub
The consultation chat is the revenue-generating core of HealConnect. A per-minute billing engine fires every 60 seconds during an active session — meaning the real-time layer must be reliable, low-latency (<200ms), and horizontally scalable.
Socket.IO provides the event-driven chat model (message:send, session:start, session:end, billing:tick) with automatic reconnection and fallback to long-polling for poor networks — critical in India's variable 4G conditions.
Azure Web PubSub acts as a managed WebSocket service in front of the Socket.IO server, allowing multiple Node.js instances to share WebSocket connections without sticky sessions or a self-managed Redis adapter.
Redis Pub/Sub routes events between Node instances in the absence of Azure Web PubSub for internal event broadcasting.

  📞  Audio & Video Calls  
6.2 Agora.io Real-Time Communication SDK
Audio and video consultations are HealConnect's second primary revenue stream. Building real-time communications infrastructure in-house is not feasible — it requires global TURN server networks, codec optimisation, NAT traversal, and years of engineering. Agora.io is the industry standard:
Global edge network with data centres in India (Mumbai, Chennai) — sub-100ms latency for Indian users.
Per-minute billing hooks are built into the Agora SDK token model, aligning perfectly with HealConnect's monetisation structure.
Network adaptive bitrate — Agora automatically adjusts call quality to available bandwidth, critical for users on variable mobile connections.
React Native SDK is officially maintained with full feature parity to the native SDKs.
Video call is deferred to Phase 3 but the Agora integration in Phase 1 (audio only) will support video with a simple channel type change — no re-architecture required.

  💳  Payments  
6.3 Razorpay (Phase 1) + Stripe (Phase 3)
Payment infrastructure is the most security-sensitive component of HealConnect. The strategy is to delegate all payment complexity to PCI-DSS Level 1 certified providers — HealConnect's servers never touch raw card data.
Razorpay dominates the Indian digital payments market with 300+ payment methods: UPI, Razorpay UPI, all major Indian bank cards, net banking, Paytm, PhonePe, and mobile wallets. No other gateway offers comparable India coverage.
Razorpay Payouts API enables direct bank transfers and UPI payouts for practitioner earnings — a single API handles the entire payout workflow.
GST-compliant invoicing is built into Razorpay's infrastructure, satisfying India's tax reporting requirements without custom development.
Stripe is selected for Phase 3 international expansion: superior global card acceptance, multi-currency support, and the industry-standard developer experience for international payments.


7. Web — Admin Panel & Landing Page
The web layer serves two distinct purposes: the public-facing marketing/landing page (SEO-critical) and the internal admin panel (operation-critical). Next.js serves both from a single codebase.
  🌐  Web Framework  
7.1 Next.js 14 with App Router
Next.js is the only web framework that serves both use cases optimally within a single project:
Server-Side Rendering (SSR) and Static Site Generation (SSG) for the landing page and blog — critical for SEO indexing of practitioner categories and wellness content, the primary organic acquisition channel.
React Server Components (RSC) for the admin panel allow server-side data fetching without client-side waterfalls, making the admin dashboard fast even with complex real-time metrics.
App Router enables co-located layouts, loading states, and error boundaries — dramatically simplifying the admin panel's complex multi-section navigation.
API Routes allow lightweight backend endpoints for CMS webhooks and admin-specific operations without a separate server.


8. Infrastructure & DevOps — Microsoft Azure
Microsoft Azure (India Central region) is selected as the exclusive cloud provider for HealConnect. Azure's India data centre presence satisfies data residency requirements under India's IT Act and PDPB, while providing enterprise-grade SLAs and seamless integration between services.
  ☁️  Cloud Infrastructure  
8.1 Core Azure Services
The infrastructure is designed as a managed PaaS-first stack — minimising operational overhead so the team can focus on product development, not server administration.

  🔄  CI/CD & Developer Workflow  
8.2 CI/CD Pipeline
The deployment pipeline is automated end-to-end using GitHub Actions for build and test, and Azure DevOps Pipelines for staging and production deployment. Every commit to main triggers the pipeline:
Code push → GitHub Actions: lint (ESLint + Prettier), type check (tsc --noEmit), unit tests (Jest), build verification.
On success → Azure DevOps: build Docker image → push to Azure Container Registry → deploy to staging slot on Azure App Service.
Staging smoke tests → manual approval gate → swap staging slot to production (zero-downtime blue-green deployment).
Azure Key Vault integration injects secrets at runtime — no secrets ever stored in the pipeline configuration or Docker images.

Security is not an afterthought — it is built into the stack at every layer. HealConnect handles financial transactions, personal health data, and KYC documents, making security a first-class engineering requirement.

13. Conclusion
The HealConnect technology stack represents a deliberate, production-grade set of choices optimised for three simultaneous constraints: a 45-day MVP deadline, a professional marketplace requiring financial-grade reliability, and a growth trajectory to 100,000+ concurrent users.
The React Native + Node.js + PostgreSQL + Azure core is a proven combination used by multiple high-scale Indian fintech and marketplace applications. The team shares TypeScript across all three surfaces (mobile, backend, web), reducing cognitive overhead and enabling rapid cross-layer contribution.
Every third-party service (Agora, Razorpay, Twilio, Firebase) has been selected for its first-class India presence, official React Native SDK, and alignment with HealConnect's per-minute consultation monetisation model. No technology in this stack requires custom integration work — all are officially supported and battle-tested in production at scale.
This stack is not over-engineered for the MVP but is intentionally architected to never require a rewrite. Phase 2 and Phase 3 features (video call, AI personalisation, physical marketplace, international expansion) slot in as additive layers on top of the existing infrastructure — not replacements of it.
